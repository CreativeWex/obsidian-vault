#### Fail-fast итераторы
Большинство `java.util` коллекций (`ArrayList`, `HashMap`, [[Map]], [[List]]) — fail-fast.
- Каждая коллекция хранит счётчик `modCount`, который увеличивается при любой структурной модификации (add/remove, но не set)
- Итератор при создании запоминает `expectedModCount = modCount`
- На каждом `next()`/`remove()` итератор сверяет `modCount == expectedModCount`. Если не совпало — бросает `ConcurrentModificationException`
- Это **best-effort защита, а не гарантия**: `modCount` не volatile, проверка не атомарна — в редких случаях гонка может не выбросить исключение, хотя модификация была

> Название обманчиво: CME может возникнуть **без всякой многопоточности**, в одном потоке

**Классический пример CME в одном потоке:**
```java
List<String> list = new ArrayList<>(List.of("a", "b", "c"));
for (String s : list) {
    if (s.equals("b")) {
        list.remove(s);   // модификация коллекции прямо во время итерации
    }
}
// ConcurrentModificationException
```
Структурная модификация коллекции напрямую (не через сам итератор) во время foreach — вот и причина, при чём поток всего один. Чтобы удалить элемент безопасно во время итерации, нужно использовать `Iterator.remove()` — этот вызов **обновляет `expectedModCount`** внутри самого итератора, поэтому не триггерит исключение.

#### Fail-safe итераторы
`CopyOnWriteArrayList`, `ConcurrentHashMap` — fail-safe, исключения не бросают.
- `CopyOnWriteArrayList`: при каждой **записи** (add/remove) создаётся **новая копия** всего внутреннего массива, ссылка на массив атомарно заменяется. Итератор, созданный до записи, продолжает работать со старым массивом-снэпшотом — не видит новых изменений, но и не падает
- `ConcurrentHashMap`: итератор **weakly consistent** — это НЕ копирование структуры (в отличие от CopyOnWrite). Итератор обходит реальные bucket'ы напрямую и допускает, что во время обхода структура меняется: может увидеть часть изменений, сделанных после создания итератора, может не увидеть другую часть — но никогда не бросает исключение и не видит "битых"/половинчатых данных
