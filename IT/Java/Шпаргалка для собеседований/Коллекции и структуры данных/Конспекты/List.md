#### ArrayList
+ хранит любые значения, даже null.
+ удаление из конца O(1), вставка в конец за O(1) или за O(n) при перестройке
+ `+` доступ к элементам по индексу за O(1), по значению за O(n)  
+ `-` вставка / удаление элемента в середину списка за O(n), вызывает перезапись всех элементов
- `-` поиск за O(n)
- `-` вставка в начало и удаление из начала за O(n) из-за перестройки;

**Resize (важная деталь, часто путают):** массив под капотом перестраивается **не при заполнении на половину**, а когда он **полностью заполнен** (`size == elementData.length`). При этом создаётся новый массив большего размера (обычно `×1.5`, `newCapacity = oldCapacity + (oldCapacity >> 1)`) и все элементы копируются — это O(n). Настроить можно только **начальную** ёмкость через конструктор (`new ArrayList<>(capacity)`), порог самого срабатывания resize не настраивается (в отличие от `loadFactor` у `HashMap`, с которым это часто путают).
#### LinkedList
- двусвязный список, каждый элемент содержит ссылки на предыдущий и следующий
- Не требует сдвига элементов при изменениях
- `+` быстрая вставка и удаление в начале и конце — O(1)
- `-` Медленный доступ по индексу — O(n)
- `-` Повышенные расходы памяти (хранение ссылок)
- `-` Вставка и удаление в середину за O(n)

#### Почему LinkedList почти всегда хуже ArrayList на практике (даже там, где Big-O должен выигрывать)
Дело не в стоимости самой перелинковки узлов (она и правда O(1) и дешёвая) — дело в **расположении в памяти**:
- `ArrayList` — один сплошной блок памяти. Элементы лежат подряд → хорошая **cache locality**: CPU подгружает данные кэш-линиями (обычно 64 байта) и может **предсказывать и предзагружать (prefetch)** следующие элементы при последовательном обходе
- `LinkedList` — каждый узел (`Node`) это отдельный объект, разбросанный по куче в произвольных местах. При обходе/доступе почти каждый переход по ссылке `next`/`prev` — это **cache miss** (нужно лезть в основную память, которая в десятки раз медленнее кэша), плюс оверхед памяти на сам объект `Node` (заголовок объекта + 2 ссылки на каждый элемент)
- Итог: даже в операциях, где `LinkedList` теоретически O(1) (например, вставка в начало), на практике она часто проигрывает `ArrayList` из-за накладных расходов на cache miss и аллокацию объектов — особенно для небольших/средних коллекций
- Альтернатива `LinkedList` для очереди/стека — `ArrayDeque`, который под капотом массив и почти всегда эффективнее
