- **Pre-training (Предварительное обучение):** самообучение на гигантских массивах текста (терабайты данных) для понимания структуры языка и накопления знаний о мире.
- **Supervised Fine-Tuning (SFT):** дообучение на отобранных вручную примерах диалогов «вопрос-ответ» для формирования стиля помощника.
- **RLHF (Обучение с подкреплением на основе отзывов людей):** использование системы оценок от людей для тренировки модели вознаграждения и финальной корректировки ответов модели
#### I. PreTraining:
###### 1. Подготовка данных и токенизация
- **Сбор данных:** Для обучения используется текстовый корпус (например, «tiny Shakespeare», содержащий около 1 млн символов).
- **Создание словаря:** Определяется набор всех уникальных символов (vocabulary), встречающихся в тексте (в примере — 65 символов).
- **Токенизация:** Реализуется перевод символов в целые числа и обратно с помощью таблиц поиска (lookup tables).
- **Разделение данных:** Данные разбиваются на обучающую (90%) и валидационную (10%) выборки для контроля переобучения.
###### 2. Формирование входных данных
- **Определение блоков:** Текст разбивается на чанки фиксированной длины (**block size** или context length), чтобы модель училась предсказывать следующий символ на основе контекста.
- **Батчинг:** Данные объединяются в пакеты (**batches**) для эффективной параллельной обработки на GPU.
###### 3. Построение базовой модели (Bigram)
- **Эмбеддинги токенов:** Создается таблица, где каждому токену соответствует вектор признаков.
- **Функция потерь:** Используется **Cross Entropy** (кросс-энтропия) для оценки качества предсказаний модели.
- **Генерация:** Реализуется функция, которая итеративно предсказывает следующий токен и добавляет его к последовательности.
###### 4. Механизм Self-Attention (Самониммание)
- **Query, Key, Value:** Каждому токену сопоставляются три вектора: **Query** (что я ищу), **Key** (что я содержу) и **Value** (что я передаю).
- **Матрица весов (Affinity):** Рассчитывается через скалярное произведение Query и Key, определяя «интерес» токенов друг к другу.
- **Маскирование (Triangular Mask):** В декодере применяется маска, чтобы текущий токен не мог «заглядывать в будущее» и общался только с прошлым контекстом.
- **Масштабирование:** Результаты скалярного произведения делятся на корень из размерности головы (Scaled Attention) для стабильности при инициализации.
###### 5. Расширение архитектуры Трансформера
- **Multi-Head Attention:** Запускается несколько голов внимания параллельно, чтобы модель могла собирать разные типы информации одновременно.
- **Feed-Forward Network:** Добавляется полносвязный слой после внимания, чтобы каждый токен мог «обдумать» собранную информацию индивидуально.
- **Позиционное кодирование:** Добавляются эмбеддинги позиций, так как механизм внимания сам по себе не знает о расположении токенов в пространстве.
###### 6. Оптимизация и глубокое обучение
- **Residual Connections:** Внедряются «скип-коннекты» (остаточные связи), создающие «супермагистраль» для градиентов, что позволяет обучать очень глубокие сети.
- **Layer Normalization:** Применяется нормализация слоев (обычно перед трансформациями) для стабилизации обучения.
- **Dropout:** Используется для регуляризации, случайно отключая часть нейронов во время обучения для предотвращения переобучения.
###### 7. Масштабирование и финальное обучение
- **Увеличение параметров:** Настройка гиперпараметров (количество слоев `n_layer`, размер эмбеддинга `n_embd`, количество голов `n_head`).
- **Цикл обучения:** Использование оптимизатора (например, **AdamW**) для обновления весов модели на протяжении тысяч итераций
